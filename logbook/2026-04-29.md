## LittleFS Persistence Test

Tested LittleFS persistence on ESP32 using a file where the app appends one `O`
on every run.

Important finding: after unplugging/replugging USB power, the app runs
immediately on power-up, before I attach `idf.py monitor`. This caused a 2-byte
jump even if I run the app just one time.

## Crystal frequency

Flashing to the ESP32 showed this warning:
`Warning: Detected crystal freq 15.55 MHz is quite different to normalized freq
26 MHz.`

To solve it, I run this to understand the crystal frequency:
`esptool.py flash_id`

Then:
`Component config → Hardware Settings → Main XTAL Config → Main XTAL frequency`

Even though the XTAL was already correct, I had to re-enter the 40MHz entry to
solve the warning.
